# Root `src/` TypeScript Migration + `commit` Pilot — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

> **Deviation from original plan:** The root tsup config is named `tsup.root.config.ts`, not `tsup.config.ts`. tsup auto-discovers any `tsup.config.ts` in a parent of the cwd, so a generic name at the repo root would leak `bundle: false` into every workspace build. The distinctive filename is invoked explicitly via `tsup --config tsup.root.config.ts`. Follow this filename literally in subsequent migration stages.

**Spec:** `docs/superpowers/specs/2026-05-20-src-typescript-migration-commit-pilot-design.md`

**Goal:** Replace the in-place root build (which commits `.ts`, `.js`, and `.d.ts` side by side in `src/`) with a conventional `src/` → `dist/` pipeline, and absorb the `packages/commit/` workspace into `src/modules/commit/` + `src/commands/commit.md` to establish the pattern for future monorepo consolidation.

**Architecture:** Two atomic commits on `feature/harness`. Commit 1 is pure infrastructure (switch root build from `tsc` to `tsup --bundle=false`, output to `dist/`, update test/script expectations, no plugin code moves). Commit 2 is the pilot absorption (move `commit` source + asset + tests into root, delete the workspace, update lockfile, add a build-output regression test). `bundle: false` is intentional to preserve current load-time semantics and avoid double-bundling workspace `dist/` content.

**Tech Stack:** TypeScript 5.8, tsup 8.5 (replaces tsc for root), vitest 3.1, npm workspaces (1 workspace removed), Node ESM (NodeNext resolution), `@opencode-ai/plugin` 1.14.

- [ ] **Step 2: Create `scripts/copy-root-assets.mjs`**

Write to `scripts/copy-root-assets.mjs`:
```js
#!/usr/bin/env node
/**
 * Copies markdown assets from src/ to dist/ preserving relative paths.
 * Used by the root build pipeline after tsup runs.
 */
import { existsSync, mkdirSync, readdirSync, statSync, copyFileSync } from "node:fs"
import path from "node:path"
import { fileURLToPath } from "node:url"

const repoRoot = path.resolve(path.dirname(fileURLToPath(import.meta.url)), "..")
const sourceRoots = ["commands", "agents", "skills"]

function copyMarkdownRecursive(sourceDir, destDir) {
  if (!existsSync(sourceDir)) return

  mkdirSync(destDir, { recursive: true })

  for (const entry of readdirSync(sourceDir)) {
    const sourcePath = path.join(sourceDir, entry)
    const destPath = path.join(destDir, entry)
    const stats = statSync(sourcePath)

    if (stats.isDirectory()) {
      copyMarkdownRecursive(sourcePath, destPath)
    } else if (entry.endsWith(".md")) {
      copyFileSync(sourcePath, destPath)
    }
  }
}

for (const root of sourceRoots) {
  copyMarkdownRecursive(
    path.join(repoRoot, "src", root),
    path.join(repoRoot, "dist", root),
  )
}
```

No build/run yet — this is verified end-to-end in Task 5.
